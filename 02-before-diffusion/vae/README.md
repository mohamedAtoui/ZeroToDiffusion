# Variational Autoencoder (VAE) on MNIST

## The Generative Story

We assume data $\mathbf{x}$ is generated by a two-step process:

1. Sample a latent variable from the prior: $\mathbf{z} \sim p(\mathbf{z}) = \mathcal{N}(\mathbf{0}, \mathbf{I})$
2. Sample the observation from the likelihood: $\mathbf{x} \sim p_\theta(\mathbf{x} | \mathbf{z})$

**Goal**: learn parameters $\theta$ to maximize the marginal likelihood:

$$p_\theta(\mathbf{x}) = \int p_\theta(\mathbf{x} | \mathbf{z}) \, p(\mathbf{z}) \, d\mathbf{z}$$

**Problem**: this integral is intractable -- we'd need to integrate over every possible $\mathbf{z}$.

**Solution**: introduce an approximate posterior $q_\phi(\mathbf{z} | \mathbf{x})$ (the encoder) and optimize the Evidence Lower Bound (ELBO) instead.

## Variational Inference & the ELBO

Starting from the log marginal likelihood, we can derive:

$$\log p(\mathbf{x}) \geq \mathbb{E}_{q_\phi(\mathbf{z}|\mathbf{x})} \left[ \log p_\theta(\mathbf{x} | \mathbf{z}) \right] - D_{\mathrm{KL}} \left( q_\phi(\mathbf{z} | \mathbf{x}) \| p(\mathbf{z}) \right)$$

The right-hand side is the **ELBO** (Evidence Lower BOund). It decomposes into two terms:

- **Reconstruction loss**: $\mathbb{E}_{q_\phi(\mathbf{z}|\mathbf{x})} \left[ \log p_\theta(\mathbf{x} | \mathbf{z}) \right]$ -- how well can the decoder reconstruct $\mathbf{x}$ from a sampled $\mathbf{z}$?
- **KL regularization**: $D_{\mathrm{KL}} \left( q_\phi(\mathbf{z} | \mathbf{x}) \| p(\mathbf{z}) \right)$ -- keep the approximate posterior $q_\phi(\mathbf{z} | \mathbf{x})$ close to the prior $\mathcal{N}(\mathbf{0}, \mathbf{I})$.

In practice we minimize the negative ELBO:

$$\mathcal{L} = -\text{ELBO} = \text{reconstruction\_loss} + \text{KL\_loss}$$

## Architecture (MNIST)

**Why MLP (not CNN)**: MNIST is simple enough that a fully-connected network works well. This keeps the focus on the VAE math rather than convolution mechanics.

**Encoder**:
```
784 -> 512 (ReLU) -> 256 (ReLU) -> two heads:
    mu       (latent_dim)
    log_var  (latent_dim)
```

**Decoder**:
```
latent_dim -> 256 (ReLU) -> 512 (ReLU) -> 784 (Sigmoid)
```

- Default `latent_dim = 20`, but try `dim=2` for visualization (see below).
- The encoder outputs **two** vectors: the mean $\boldsymbol{\mu}$ and log-variance $\log \boldsymbol{\sigma}^2$ of $q_\phi(\mathbf{z} | \mathbf{x})$.
- The Sigmoid on the decoder output squashes pixels to $[0, 1]$, matching the normalized MNIST range.

## The Reparameterization Trick

**Problem**: we need to sample $\mathbf{z} \sim q_\phi(\mathbf{z} | \mathbf{x}) = \mathcal{N}(\boldsymbol{\mu}, \boldsymbol{\sigma}^2 \mathbf{I})$, but sampling is not differentiable -- we can't backpropagate through a random draw.

**Solution**: reparameterize the sampling as a deterministic function of the parameters plus external noise:

$$\mathbf{z} = \boldsymbol{\mu} + \boldsymbol{\sigma} \odot \boldsymbol{\epsilon}, \quad \boldsymbol{\epsilon} \sim \mathcal{N}(\mathbf{0}, \mathbf{I})$$

This moves the stochasticity to $\boldsymbol{\epsilon}$, making $\mathbf{z}$ differentiable with respect to $\boldsymbol{\mu}$ and $\boldsymbol{\sigma}$ (and thus $\phi$).

In code:
```python
z = mu + torch.exp(0.5 * log_var) * torch.randn_like(mu)
```

Note: we store $\log \sigma^2$ (i.e., `log_var`) instead of $\sigma$ directly for numerical stability, and recover $\sigma = \exp(0.5 \cdot \log \sigma^2)$.

## Loss Function

**Reconstruction loss** -- Binary Cross-Entropy summed over all 784 pixels:

$$\mathcal{L}_{\text{recon}} = -\sum_{i=1}^{784} \left[ x_i \log \hat{x}_i + (1 - x_i) \log (1 - \hat{x}_i) \right]$$

where $\hat{x}_i$ is the decoder output (Sigmoid-activated).

**KL divergence** -- closed-form solution for Gaussian $q_\phi(\mathbf{z}|\mathbf{x})$ vs. $\mathcal{N}(\mathbf{0}, \mathbf{I})$ prior:

$$\mathcal{L}_{\text{KL}} = -\frac{1}{2} \sum_{j=1}^{d} \left( 1 + \log \sigma_j^2 - \mu_j^2 - \sigma_j^2 \right)$$

In code:
```python
kl_loss = -0.5 * torch.sum(1 + log_var - mu.pow(2) - log_var.exp())
```

**Total loss**:

$$\mathcal{L} = \mathcal{L}_{\text{recon}} + \beta \cdot \mathcal{L}_{\text{KL}}$$

where $\beta$ is the KL weight. With **KL annealing**, $\beta$ is linearly increased from 0 to 1 over the course of training.

## Known Issues & What to Watch For

- **Blurry samples**: VAEs optimize a pixel-wise reconstruction loss (BCE), which encourages the model to hedge its bets and produce averaged/blurry outputs. This is a fundamental limitation compared to GANs or diffusion models.
- **Posterior collapse**: the KL term can dominate early in training, pushing $q_\phi(\mathbf{z}|\mathbf{x})$ to match $p(\mathbf{z})$ exactly. When this happens the encoder ignores $\mathbf{x}$ entirely and $\mathbf{z}$ carries no information.
- **KL annealing**: gradually increasing the KL weight $\beta$ from 0 to 1 gives the reconstruction term time to learn useful features before the KL penalty kicks in, helping avoid posterior collapse.
- **Latent dim tradeoff**: too small (e.g., 2) = underfitting, the model can't represent all variation; too large (e.g., 200) = many dimensions go unused and collapse to the prior.
